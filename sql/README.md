洛克王国宠物图鉴数据库（pokedex_db）使用说明
一、项目简介
本项目为洛克王国宠物图鉴系统专用数据库设计，包含完整的表结构创建、初始数据填充脚本，支持用户管理、宠物信息、宠物属性、技能体系、用户背包、宠物技能关联等核心业务场景，基于 MySQL 5.7+ / 8.0+ 开发，字符集采用 utf8mb4 支持全字符存储。
二、文件说明
项目仅包含两个核心 SQL 文件（合并为完整可执行脚本），功能分工：
建表 + 初始化数据脚本
包含：数据库创建、7 张业务表创建、外键约束、索引优化、全量初始数据插入
覆盖：用户数据、67 只宠物基础信息、宠物种族值、15 个技能、宠物 - 技能关联、用户背包数据
三、数据库表结构设计
共 7 张核心表，关系清晰，满足宠物图鉴系统业务需求：
表格
表名	表用途	关联关系
users	用户信息表	核心主表，存储用户账号、昵称、状态等
pokemon	宠物基础信息表	核心主表，存储宠物编号、名称、属性、形象等
pokemon_stats	宠物种族值表	与 pokemon 一对一关联，存储六围属性
skill	技能信息表	核心主表，存储技能名称、类型、威力、效果
pokemon_skill_relation	宠物 - 技能关联表	宠物与技能多对多关联，记录学习等级
user_pokemon	用户背包表	用户与宠物多对多关联，记录拥有的宠物
四、核心功能支持
用户体系：支持账号密码、昵称、头像、登录状态、禁用 / 正常状态管理
宠物管理：67 只洛克王国宠物全覆盖，支持编号、名称、属性、身高体重、简介、形象、稀有 / 神兽标记
属性体系：宠物六围种族值（生命值、攻击、防御、魔攻、魔抗、速度）
技能体系：15 个经典技能，区分物理 / 魔法 / 变化类型，记录技能效果
养成关联：宠物可学习多个技能，用户可拥有多只宠物
数据约束：外键级联删除、唯一约束、索引优化，保证数据完整性与查询效率
五、执行步骤
环境要求
MySQL 5.7 / 8.0 及以上版本
支持 InnoDB 存储引擎
执行方式
打开 MySQL 客户端（Navicat、DBeaver、命令行等）
复制完整 SQL 脚本，直接执行（脚本已包含 CREATE DATABASE，无需手动建库）
执行完成后，刷新数据库，即可看到 pokedex_db 库及所有表和数据
注意事项
脚本已内置 IF NOT EXISTS，重复执行不会报错
内置 SET FOREIGN_KEY_CHECKS = 0，清空数据时无外键冲突
初始密码为 BCrypt 加密格式，默认用户：
管理员：admin / 加密密码
普通用户：xiaoluoke / 加密密码
六、初始数据概览
用户数据：2 条（管理员 + 普通用户）
宠物数据：67 只（草 / 火 / 水 / 冰 / 电 / 幽灵 / 石 / 武 / 光 / 翼 / 虫 / 恶魔 / 龙 13 系）
种族值：67 条，对应每只宠物六围属性
技能数据：15 个经典战斗技能
关联数据：宠物 - 技能关联、用户 - 宠物关联完整匹配
七、适用场景
洛克王国宠物图鉴 Web / 小程序 / APP 后端数据库
宠物养成类游戏学习项目
MySQL 表设计、外键、多对多关系实战案例
八、技术亮点
规范命名：表名、字段名语义化，注释完整
数据安全：密码加密存储、状态控制、外键级联保护
性能优化：常用字段建立索引，提升查询速度
